# OpenCode Output Path

Padrão único de diretório para **todas as operações de saída** do opencode.
Nada deve ser salvo em `C:\` ou em diretórios temporários — tudo vai para `D:\Opencode\`.

## Regra geral

| Tipo | Caminho | Criação |
|---|---|---|
| Projetos analisados | `D:\Opencode\{project-name}\` (WSL: `/mnt/d/Opencode/{project-name}/`) | `mkdir -p` automático |
| Downloads avulsos | `D:\Opencode\Download\` (WSL: `/mnt/d/Opencode/Download/`) | `mkdir -p` automático |
| Clonagem de repositórios | `D:\Opencode\{project-name}\` | `git clone <url> "/mnt/d/Opencode/{project-name}"` |
| Documentos gerados | `D:\Opencode\{project-name}\docs\` | `mkdir -p` automático |
| Relatórios JSON | `D:\Opencode\{project-name}\*.json` | `mkdir -p` antes de salvar |

## Versionamento automático

**Nenhum arquivo é sobrescrito.** Todo output recebe um prefixo com data e versão incremental:

```
{YYYY-MM-DD}_v{N}_{nome-original}
```

### Regras

- **Data**: extraída do sistema no momento da geração (`date +%Y-%m-%d`)
- **Versão**: contador inteiro que inicia em `1` e incrementa a cada novo arquivo gerado no **mesmo dia e diretório**
- O contador analisa TODOS os arquivos/pastas existentes que começam com `{YYYY-MM-DD}_v` para calcular a próxima versão
- Ao mudar o dia, o contador reinicia em `v1`
- Se um arquivo com o mesmo nome for gerado de novo no mesmo dia, ganha um número de versão **maior** (nunca substitui o anterior)

### Exemplos

```
/mnt/d/Opencode/meu-projeto/2026-07-08_v1_relatorio.pdf
/mnt/d/Opencode/meu-projeto/2026-07-08_v2_relatorio.pdf   ← reanálise no mesmo dia
/mnt/d/Opencode/meu-projeto/2026-07-08_v1_analise.json
/mnt/d/Opencode/meu-projeto/2026-07-09_v1_relatorio.pdf   ← dia seguinte
```

## Como usar

### Caminho no WSL (Linux)

```
/mnt/d/Opencode/{project-name}/
```

### Caminho no Windows

```
D:\Opencode\{project-name}\
```

### Script auxiliar

```bash
# Apenas criar diretório (retorna o caminho)
bash ~/.config/opencode/skills/opencode-output-path/ensure.sh "meu-projeto"
# → /mnt/d/Opencode/meu-projeto

# Obter prefixo versionado (cria diretório se necessário)
bash ~/.config/opencode/skills/opencode-output-path/ensure.sh "meu-projeto" --v
# → /mnt/d/Opencode/meu-projeto/2026-07-08_v1

# Obter caminho completo com nome de arquivo versionado
bash ~/.config/opencode/skills/opencode-output-path/ensure.sh "meu-projeto" --v "relatorio.pdf"
# → /mnt/d/Opencode/meu-projeto/2026-07-08_v1_relatorio.pdf

# Sem argumento → Download
bash ~/.config/opencode/skills/opencode-output-path/ensure.sh
# → /mnt/d/Opencode/Download
```

## Fluxo obrigatório para QUALQUER saída

1. Identificar o nome do projeto (ou usar "Download" para avulsos)
2. Executar: `ensure.sh "projeto" --v "arquivo.ext"` para obter o caminho versionado
3. Salvar o output no caminho retornado
4. Confirmar para o usuário onde o arquivo foi salvo

## Exemplos completos

```bash
# Relatório versionado
PATH=$(bash ~/.config/opencode/skills/opencode-output-path/ensure.sh "meu-projeto" --v "relatorio.pdf")
weasyprint input.html "$PATH"

# JSON de análise versionado
PATH=$(bash ~/.config/opencode/skills/opencode-output-path/ensure.sh "meu-projeto" --v "scanner-output.json")
scanner --output "$PATH"

# Download versionado
PATH=$(bash ~/.config/opencode/skills/opencode-output-path/ensure.sh "Download" --v "imagem.png")
wget -O "$PATH" "https://exemplo.com/img.png"
```

## Interdições

- **NUNCA** salve em `C:\Users\...`, `C:\Temp\`, `~`, `/tmp`, ou diretórios do projeto sendo analisado
- **SEMPRE** use `/mnt/d/Opencode/` como raiz
- **NUNCA** crie arquivos fora de `D:\Opencode\` sem permissão explícita do usuário
- **NUNCA** sobrescreva um arquivo existente — SEMPRE use o prefixo versionado
